📘 **Módulo: Persistência de Dados com LocalStorage**

🎯 **O que vamos aprender?**
O objetivo é criar o fluxo completo de cadastro. Você vai aprender a interceptar o envio de um formulário, capturar os valores dos campos de texto manualmente, organizar esses dados em um objeto e guardá-los no "armário" do navegador (`localStorage`).

💡 **Teoria Aplicada**

### O Fluxo de Salvamento

Como ainda não estamos usando ferramentas automáticas (como o `FormData`), faremos o processo de "curadoria" manual dos dados:

1. **Interceptação:** Usamos `e.preventDefault()` para o navegador não recarregar a página.
2. **Captura:** Selecionamos os inputs pelo ID e pegamos o seu `.value`.
3. **Agrupamento:** Criamos um objeto `{ }` para manter o título e o autor juntos.
4. **Serialização:** Como o `localStorage` só guarda texto, usamos `JSON.stringify()` para transformar o objeto em uma string.
5. **Armazenamento:** Salvamos na chave correta.

```javascript
// Exemplo técnico de captura manual
const tituloInformado = document.querySelector('#input-titulo').value;
const autorInformado = document.querySelector('#input-autor').value;

const novoLivro = {
    titulo: tituloInformado,
    autor: autorInformado
};
```

🎬 **No Cenário da BookTuber**

Pense nisso como o backup do seu canal. Se você escreve um roteiro de resenha e a luz acaba, você quer que o texto esteja lá quando o computador ligar. O `localStorage` é o seu "cartão de memória" onde o JavaScript salva cada livro da Sally Rooney que você decide adicionar à sua estante virtual.

✍️ **Exercícios de Fixação**

### 1. O Script de Inicialização (`init`)

> Scripts que usem LocalStorage ou document.addEventListener só podem ser executados no Browser e não em NodeJS

No seu arquivo `app.js`, crie uma função chamada `init()`.

* Dentro dela, coloque apenas um `console.log("Sistema BookSelf carregado e pronto!")`.
* Configure o evento `DOMContentLoaded` para chamar essa função `init`.
* *Por que isso é útil?* Isso garante que qualquer tentativa de buscar o formulário só aconteça quando o HTML estiver 100% pronto.

### 2. Dando Vida à Estante: Cadastro de Livros

> Scripts que usem LocalStorage ou document.addEventListener só podem ser executados no Browser e não em NodeJS

Agora, vamos criar a lógica de salvamento dentro da sua função `init`.

**O Desafio:**

1. Selecione o formulário pelo ID.
2. Adicione um `addEventListener` de `'submit'`.
3. Dentro da função do evento:
    * Use o `preventDefault()` para evitar o reload.
    * Capture o valor do campo **Título** e do campo **Autor**.
    * Crie um objeto chamado `livro`.
    * Salve esse objeto no `localStorage` sob a chave `'ultimo_livro_cadastrado'`. (Lembre-se de usar `JSON.stringify`).

### 3. Pequena Validação de Segurança

> Scripts que usem LocalStorage ou document.addEventListener só podem ser executados no Browser e não em NodeJS

Modifique o exercício anterior: o código só deve salvar no `localStorage` se ambos os campos (Título e Autor) estiverem preenchidos. Caso contrário, exiba um `alert("Ei, você esqueceu de preencher os dados do livro!")`.
